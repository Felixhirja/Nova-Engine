# Nova-Engine Technical Roadmap

*** C++17 3D GAME ENGINE - MODULAR ARCHITECTURE ***

_Last updated: November 3, 2025_

---

## Engine Overview

**Nova-Engine** is a modern C++17 3D game engine designed for space simulation games, featuring:
- High-performance ECS architecture with archetype-based storage
- Data-driven configuration system with hot-reload
- OpenGL rendering pipeline with modern shader support
- Cross-platform support (Windows, Linux, macOS planned)
- Modular design for extensibility

**Current Version:** 0.9 (Pre-Release)
**Target Version 1.0:** Q2 2026

## PHASE 1: RENDERING MODERNIZATION (CRITICAL - Q4 2025)

### OpenGL 4.6 Core Profile Migration

**Priority: CRITICAL (Blocks current GLFW crash)**

□ **Immediate Actions**
  □ Update GLFW context creation to request OpenGL 4.6 Core
  □ Add GLAD for modern OpenGL function loading
  □ Implement fallback chain (4.6 → 4.5 → 4.3 → 3.3 → 2.1)
  □ Add GPU capability detection and logging
  □ Create graphics_capabilities.log on startup

□ **Legacy Rendering Removal**
  □ Eliminate all glBegin/glEnd immediate mode calls
  □ Replace with VAO/VBO/EBO rendering
  □ Update all geometry to use modern vertex attributes
  □ Remove deprecated glVertex/glColor/glTexCoord calls

□ **Modern Rendering Pipeline**
  □ Mesh class with VAO/VBO/EBO management
  □ Material system with uniform buffers (UBOs)
  □ Shader program management and caching
  □ Texture unit management
  □ Render state management

□ **Advanced Features**
  □ Instanced rendering for repeated objects
  □ Compute shaders for GPU-driven processing
  □ Tessellation shaders for LOD
  □ Geometry shaders for effects
  □ Multi-draw indirect for batch rendering

**Estimated Effort:** 70-130 hours (2-3 weeks)
**Impact:** Fixes GLFW crash, enables modern GPU features, 5-10x rendering performance

## Performance Targets

### Engine Performance Goals

**Rendering:**
- 60+ FPS at 1080p on mid-range hardware (GTX 1060 / RX 580)
- 144+ FPS at 1080p on high-end hardware (RTX 3070 / RX 6800)
- Support for 4K rendering at 60 FPS on modern GPUs

**Memory:**
- <2GB base engine memory footprint
- Efficient texture streaming (budget-based)
- No memory leaks (verified with sanitizers)

**Loading:**
- Scene load time <3 seconds for medium scenes
- Asset streaming with <16ms frame budget
- Hot-reload response time <100ms

**Physics:**
- 1000+ rigid bodies at 60Hz simulation
- Sub-frame collision detection
- Deterministic simulation for networking

**ECS:**
- 100,000+ entities with acceptable performance
- Sub-millisecond query times for filtered queries
- Parallel system execution scaling with CPU cores
